## 서비스 개요

**제품명**: Invoice Web Viewer
**한 줄 설명**: 노션(Notion)에서 작성한 견적서를 클라이언트가 웹 링크로 확인하고 PDF로 다운로드할 수 있는 서비스

**핵심 사용자 시나리오**:
1. 프리랜서/에이전시 담당자(Admin)가 노션 데이터베이스에 견적 항목을 입력한다.
2. 시스템이 견적서 고유 URL을 생성하여 Admin에게 제공한다.
3. Admin이 해당 URL을 클라이언트(Client)에게 공유한다.
4. Client가 URL에 접속하여 브라우저에서 견적서를 확인한다.
5. Client가 [PDF 다운로드] 버튼을 눌러 견적서를 저장한다.

**기술 스택 (고정)**:
- 프레임워크: Next.js App Router (서버 컴포넌트 우선)
- UI: shadcn/ui + Tailwind CSS v4
- 폼: React Hook Form + Zod
- 데이터 소스: Notion API (노션 데이터베이스)
- PDF 생성: 브라우저 출력(window.print) 또는 puppeteer/react-pdf 중 MVP에 적합한 방식
- 인증: 없음(MVP 범위 - URL 기반 접근)

## PRD 작성 지침

다음 목차 구조를 반드시 포함하여 마크다운 형식으로 작성하세요.

### 1. 문서 메타데이터
- 버전, 작성일, 작성자(예시), 상태(Draft/Review/Approved)

### 2. 제품 목적 및 배경
- 해결하려는 문제
- 목표 사용자(Persona): Admin, Client 각각 1~2문장
- 비즈니스 목표(정성적)

### 3. 범위 정의
- **MVP In-Scope**: 이번 버전에서 반드시 구현할 기능
- **MVP Out-of-Scope**: 이번 버전에서 제외할 기능 (이유 포함)

### 4. 기능 요구사항

각 기능은 다음 형식으로 작성하세요:

```
#### F-[번호]. [기능명]
- **우선순위**: P0(필수) / P1(높음) / P2(보통)
- **설명**: 기능 설명
- **사용자 스토리**: "나는 [역할]로서, [행동]을 원한다. 왜냐하면 [이유]이기 때문이다."
- **인수 조건(AC)**:
  - [ ] AC-1: ...
  - [ ] AC-2: ...
- **기술 구현 힌트**: Next.js/Notion API 관점에서 간략한 구현 방향
```

반드시 포함할 기능 목록 (F-001부터 순서대로):
1. 노션 데이터베이스 연동 및 견적 데이터 조회
2. 견적서 고유 URL 생성 (슬러그 또는 UUID 기반)
3. 견적서 웹 뷰어 페이지 (공개 접근)
4. 견적서 항목 표시 (품목, 수량, 단가, 금액, 합계, 세금, 최종금액)
5. PDF 다운로드 기능
6. 견적서 상태 표시 (초안/발송됨/승인됨/거절됨)
7. Admin 대시보드 - 견적서 목록 조회
8. Admin 대시보드 - 공유 링크 복사

### 5. 비기능 요구사항
- **성능**: 견적서 페이지 LCP 기준
- **보안**: URL 기반 접근의 보안 고려사항
- **반응형**: 모바일/데스크탑 지원 여부
- **접근성**: 기본 WCAG 준수 항목

### 6. 데이터 모델 (Notion 데이터베이스 스키마)
노션 데이터베이스의 속성(Property) 구조를 테이블 형식으로 정의하세요.

| 속성명 | Notion 타입 | 설명 | 필수 여부 |
|--------|------------|------|----------|
| ... | ... | ... | ... |

### 7. 페이지 및 라우트 구조

Next.js App Router 기준으로 라우트를 정의하세요.

```
app/
├── (marketing)/           # 랜딩 페이지
├── (auth)/                # Admin 로그인(Out-of-Scope)
├── (dashboard)/           # Admin 대시보드
│   └── invoices/
│       └── [id]/
└── invoice/               # 공개 견적서 뷰어
    └── [slug]/
```

각 라우트에 대해:
- URL 패턴
- 접근 권한
- 주요 컴포넌트

### 8. UX 흐름도
Admin 흐름과 Client 흐름을 각각 Mermaid 다이어그램으로 작성하세요.

### 9. 오픈 이슈 및 의사결정 필요 항목
- MVP 범위에서 결정이 필요한 항목을 테이블로 정리
- 각 항목에 대한 권장 옵션 제시

### 10. 마일스톤 (예시)
MVP 완성까지의 단계를 3~4개의 마일스톤으로 구성하세요.
각 마일스톤에 포함될 기능(F-번호)을 명시하세요.
